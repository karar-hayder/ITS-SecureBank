# Contributing to SECUREBANK

We welcome contributions! Please follow these guidelines to ensure a smooth collaboration process.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:

   ```bash
   git clone https://github.com/YOUR_USERNAME/ITS-Hackthon-Backend.git
   ```

3. **Create a new branch** for your feature or bugfix:

   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Environment

- **.NET 8 SDK** is required.
- **PostgreSQL** for the database.
- Follow the setup instructions in the [README.md](README.md).

## Coding Standards

- Follow C# standard naming conventions.
- Ensure all public methods have XML documentation if applicable.
- Write unit tests for new business logic in `Backend.Tests`.

## Pull Request Process

1. Ensure your code builds and all tests pass.
2. Update documentation (`SCHEMA.md`, `WORKFLOW.md`, etc.) if your changes affect architecture or API flows.
3. Submit a Pull Request (PR) with a clear description of the changes and why they are needed.
4. Wait for review and address any feedback.
